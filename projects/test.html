project overview

classic robotics project but with a twist - instead of traditional ir sensors, i used computer vision for line detection. this approach allows for more complex path following and better handling of intersections and curves. the pid controller ensures smooth movement without oscillations.

the robot can follow black lines on white surfaces, handle 90-degree turns, navigate intersections, and avoid obstacles while maintaining course. it's designed for robotics competitions but also serves as a learning platform for computer vision and control systems.

computer vision pipeline

1

image acquisition

capture 640x480 frames at 30fps from usb camera, with automatic exposure and white balance adjustments for consistent image quality.

2

preprocessing

convert to hsv color space, apply gaussian blur to reduce noise, and use adaptive thresholding to handle varying lighting conditions.

3

line detection

use contour detection to find line boundaries, calculate line center using moments, and determine line angle for steering control.

4

path planning

calculate desired heading based on line position and angle, apply lookahead distance for smoother turns, and generate motor commands.

pid control system

proportional (p) control

responds to current error between desired and actual robot position relative to line. larger error = stronger correction.

kp value: 0.8

integral (i) control

corrects for systematic bias and steady-state errors by accumulating error over time. helps eliminate offset.

ki value: 0.02

derivative (d) control

predicts future error based on rate of change, reducing overshoot and oscillations for smoother movement.

kd value: 0.1

tuning process

used ziegler-nichols method as starting point, then fine-tuned through iterative testing on various track configurations.

update rate: 20 hz

challenges & solutions

real-time image processing for line detection

problem: processing 30fps video stream while maintaining real-time control response was computationally challenging.

solution: optimized opencv pipeline by reducing image resolution for processing, implementing region of interest (roi) to focus on relevant areas, and using efficient contour detection algorithms.

smooth pid control tuning

problem: robot exhibited oscillations and overshoot when following curved sections of the track.

solution: implemented systematic tuning approach starting with proportional gain, added derivative control to reduce overshoot, and fine-tuned using real track testing data.

handling different lighting conditions

problem: line detection accuracy dropped significantly under fluorescent lights or near windows with natural light.

solution: implemented adaptive thresholding using gaussian weighted averages, added led ring light for consistent illumination, and calibrated hsv color ranges for different environments.

obstacle avoidance while line following

problem: integrating obstacle avoidance without losing track of the line required complex state management.

solution: developed finite state machine with states for normal following, obstacle detected, avoiding obstacle, and returning to line. used combination of ultrasonic sensor and vision to maintain line awareness during avoidance.
